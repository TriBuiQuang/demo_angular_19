- Resource có 3 phần:

1. Một request function, thể hiện yêu cầu chính xác cần thực hiện theo signals. Ví dụ, một user resource có thể tính toán một request phụ thuộc vào tham số user ID trong route hiện tại.

2. Loader, thực hiện thao tác asynchronous khi yêu cầu thay đổi và cuối cùng trả về giá trị mới.

3. Resource instance nhận được sẽ cung cấp các signals để truyền tải cả giá trị (khi có sẵn) cũng như trạng thái hiện tại của resource (đang tải, đã hoàn thành, gặp lỗi, v.v.)."
